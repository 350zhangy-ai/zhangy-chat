# Zhangy Chat R3 - MiniMind 风格思考可见版

## 功能说明

此版本参考 **MiniMind-R1** 项目的推理链（CoT）展示方式，实现了思考过程可见功能。AI 回答前先显示思考过程，使用 `<think></think>` 标签包裹。

## 核心特点

1. **思考过程使用 <think></think> 标签**: 类似 MiniMind-R1 的推理链展示
2. **深色主题界面**: 左侧思考区（终端风格）+ 右侧对话区（GitHub Dark 主题）
3. **可开关思考显示**: 用户可选择是否显示思考过程
4. **完全基于 R3**: 不依赖 R4/R5 的新模块，轻量级实现

## 思考过程展示

```
<think>
  分析问题：用户输入：我饿了...
  匹配回复库：找到相关类别建议
  生成回答：从实用回复库提取
</think>

## 吃饭建议

**快速解决**:
- 点外卖：美团/饿了么
- 楼下快餐：10-15 元
- 泡面 + 蛋：5 分钟
```

## 运行方式

### GUI 模式（推荐）
```bash
python gui_r3_minimind.py
```

界面布局:
- **左侧**: 思考过程显示区（黑底绿字，终端风格）
- **右侧**: 对话区（GitHub Dark 主题）
- **底部**: 思考过程开关（可勾选）

### CMD 模式
```bash
python -c "from zhangy_chat.assistant_r3_minimind import Assistant; a = Assistant(); print(a.chat('你的问题', show_thought=True))"
```

## 思考过程类型

| 步骤 | 说明 |
|------|------|
| 分析问题 | 理解用户输入的核心诉求 |
| 识别类型 | 判断是简单对话/实用问题/逻辑推理 |
| 情感分析 | 检测用户情绪状态（如有） |
| 匹配回复库 | 查找预设的实用回复 |
| 推理启动 | 未匹配时启动逻辑推理引擎 |
| 问题类型 | 因果推理/流程推理/决策推理 |
| 推理步骤 | 分步推理过程展示 |
| 生成回答 | 基于推理生成最终回答 |

## 文件说明

| 文件 | 说明 |
|------|------|
| `assistant_r3_minimind.py` | MiniMind 风格 AI 助手核心 |
| `gui_r3_minimind.py` | MiniMind 风格 GUI 界面（深色主题） |
| `config.yaml` | 配置文件（R3 版本） |

## 与原版 R3 的区别

| 功能 | R3 原版 | R3 MiniMind 版 |
|------|--------|--------------|
| 思考过程 | 不显示 | <think></think> 标签包裹 |
| 界面主题 | 浅色 | 深色（GitHub Dark） |
| 思考区布局 | 无 | 左侧独立显示 |
| 响应速度 | 正常 | 略慢（需记录思考） |
| 适用场景 | 日常使用 | 调试/演示/学习 |

## 技术实现

1. **思考记录**: `self.thought_process` 列表记录每一步思考
2. **标签包裹**: 使用 `<think></think>` 包裹思考过程
3. **GUI 显示**: 左侧文本框实时显示思考过程，高亮标签
4. **可开关**: 通过复选框控制 `show_thought` 参数


## 注意事项

- 此版本为**独立版本**，不影响原有 R3/R4/R5 代码
- 思考过程会增加少量响应时间（约 100-200ms）
- 适合演示 AI 内部工作原理
- 日常使用可关闭思考显示，提升响应速度
